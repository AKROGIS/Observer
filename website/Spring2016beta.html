Park Observer 2016

We have been working on Park Observer this spring, and we have some nice new feature. They are still in development, so we apppreciate any testing and feedback. The features are only available on version 0.9.8 or higher. (To check the version, open the Settings App and tap on Park Observer near the bottom of the list on the left. NOTE There is a bug in the Settings app if the Park Observer settings page is mostly blank, kill the settings app (double click the home button and swipe up on the settings app) and try again.) Use the instructions below to install the latest version.

Some of the new features are:

  • Park Observer to ArcMap Tool

    Export or email your Survey to an ArcGIS computer and build a file geodatabase of all your survey data. No need to connect to the NPS network. More info...

  • Advanced Symbology

    Lines and points in Park Observer can now use most of the symbology available to ArcGIS. Observations can have different symbols and/or colors based on attributes of the observation. More info...

  • Track Log Totalizer

    Display a running time or distance by survey segment. More info...

  • Recording Indicator

    User configurable text (via the protocol file) can be displayed on the map to indicate: "Recording but not observing" and "Observing (on Transect)" Contact us, or see the Protocol File Specifications for more info.

We are also working on several other features we plan to have ready this spring:

  • Required Fields

    Specify (via the protocol file), that selected fields must not be blank (nor contain 0) in order to close the form. This feature should be used sparingly. It violates the notion that the user is in charge of the app, and puts the app in charge of the user. Contact us, or see the Protocol File Specifications for more info.

  • Unique Identifier for Observations

    Display a unique identifier for each observation at the top of the attribute entry popup. This identifier may be a timestamp or a sequential number. An option to label the observation on the map with this identifier is also under consideration.

  • Background tracklogging

    Currently the app does not get location updates if you switch to another app while recording. This can be fixed.

  • Move Buttons to the Top

    Add an option via the Settings app to allow a user to decide if the button toolbar should be at the bottom (default) or at the top of the screen.

  • Other

    The full list of bugs, feature requests and other issues is available on source code repository on github

Installing the Upgrade

Until thes new features are finished and tested, version 0.9.7 will be the default over the air install from this web site. This means you will need to install the app manually to use the latest features.

  1. Download the application package to your computer.
  2. Follow the remainder of the instructions Getting Started without Wifi.
  3. Download the Park Observer Toolbox for ArcMap. After downloading, move this file somewhere that is convenient for ArcCatalog to find.

View data in ArcMap

This feature is for those who cannot access to the Park Observer server on the NPS wireless network, and those that want to view/edit the data collected in ArcMap while still in the field.

To use this feature you will need to upgrade the app and install the Park Observer Toolbox on your ArcGIS PC. The export file (*.poz) created with older (0.9.7 or earlier) versions of Park Observer is not compatible with ArcGIS.

More complete instructions will be provided shortly (with screenshots and everything). In the meantime, the following should work for the intrepid.

  1. Install the latest version of the app, and the Park Observer Toolbox
  2. Tap on the survey button
  3. Tap on the share icon (square with up arrow) next to the survey to share
  4. Tap on Share Survey with iTunes
  5. Connect your iPad to an ArcGIS PC
  6. Copy the survey file (*.poz) found in the Park Observer File Sharing area of iTunes (see Getting Started without Wifi for help on iTunes file sharing) to a folder on your PC.
  7. Open ArcCatalog
  8. In ArcCatalog, browse to the location where you saved the Park Observer Toolbox (ParkObserver.pyt) file downloaded in the first step.
  9. Open the toolbox and double click the Survey to FGDB tool.
  10. There is only one input to this tool: the poz file from step 6
  11. When the tool is done, it will have created a FGDB in the same folder as the survey (*.poz) file. The FGDB is named to match the survey protocol name and version
  12. The FGDB will include feature classes for the tracklogs, gps points, various kinds of observations and relationships among these various feature classes. Just drag and drop all the feature classes into ArcMap and symbolize/select to your heart's content.

Advanced Symbology

Track log lines can now have differnt styles (dashed, dotted, etc). Observation points can now have different shapes, outlines, even icons. All of the renderers and options describe here are supported. The JSON description of your desired symbology must be added to your upgraded protocol file.

Specifying Unique Value and Class Break renderers is supported, however a bug in the app is keeping the features from being rendered properly. This should be fixed soon.

To use this feature you will need to upgrade the app, and upgrade your protocol files.

Transect Totalalizer

The totalizer puts text on the map in the upper left hand corner to show the duration spent under the current conditions. This text will not appear unless special instructions are included in your protocol file. The text can display the time or distance (in miles or kilometers) while observing, not observing or both. This duration is totaled for each value of a mission property attribute you select. For example if your survey collects a transect ID, you can display the distance covered so far observing on that transect line.
