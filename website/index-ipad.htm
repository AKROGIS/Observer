Park Observer

An iPad application for offline data collection

Brought to you by the Alaska Regional Office GIS Team



Getting Started

  1. Install the App

    This will install the most recent version of the app.
    See the list of downloads for details and options to install older versions.

  2. Beginning with iOS 9.0, you will not be able to run the app until the device trusts the developer. To do this open the Settings app and go to General -> Profiles & Device Management -> National Park Service -> Trust on your iPad to indicate you trust the NPS to develop enterprise apps.

  3. Enterprise apps will expire every year. You can update, by reinstalling (with the button above). Updating will preserve all of your current data and settings.

  4. Use the app

    At first the app has no map or data to display so it will look like the following:

  5. Maps

    A list of ready to use basemaps. These maps are available in Park Observer (when you are on the NPS network) via the Maps button.

  6. Protocols

    A list of ready to use survey protocols. A protocol specifies the attributes to collect at each observation and other specific requirements for a particular type of survey. These protocols are available in Park Observer (when you are on the NPS network) via the Add (+) button on the survey list.

  7. Survey Results

    A list to data (and maps) collected form Park Observer. This list only contains data that has been Synced with the server on the iPad.

  8. Creating Maps/Protocols

    Instructions on what to do if there is no protocol or basemap for your survey.
