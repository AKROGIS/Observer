Major Changes

  • Clean up styling of Status Text and make font size configurable

    The text for on/off-transect (or observing/not-observing), as well as the transect totalizer is now a clean font on an opaque gray background. The font size can be configured with options in the obsprot file, so individual surveys can control this to thier specifications.

  • Configurable control for when the Mission Properties dialog is displayed for editing

    Options can be configured in the protocol file. You can display the MP editor at any of the following times:

    1. Start Recording - default is on
    2. Start Observing (first one after recording) default is off
    3. Start Re-Observing (after stop observing) default is on
    4. Stop Observing Prior (edits the track log segment being closed) - default is off
    5. Stop Observing - default is off

    There is a limitation where only one of the last two options is allowed. If both are on, then the second one wins. If you want to edit the prior track log segment, you can tap the marker on the map at the start of the segment.

  • An alert is presented when deleting a feature

    Deletion is an undoable action. Since it is possible, especially in a bumpy aircraft, to accidentally tap the delete button when reviewing or editing an existing observation, the inconvenience of agreeing to deletion seems warranted.

  • Allow 'off-transect' observations

    Off-transect is synonymous with not observing, and on-transect equals observing. While it seems illogical to allow observations when you are not observing, there are some surveys that want to be able to make observations at any time, regardless of the on/off-transect setting. This feature is enabled in the protocol file on a feature by feature basis. The default is off.

  • Increase spacing around the Mission Properties (partly cloudy) button

    This helps to avoid mistaken taps.

See the protocol specification file for details on enabling the configurable options.
