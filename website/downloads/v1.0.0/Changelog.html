Change Log for version 1.0.0

The definitive list of changes is tracked in the GitHub Repo

  • Issues closed since previous release
  • Code commits since previous release

Major Changes

  • Attributes can be entered when you start recording

    Previously, the mission property attributes only comes up when you start observing, or change environment, it should also happen when you start recording. Previously, transects all start with a small segment on the wrong transect id, because the default when recording starts is the prior transect id.

  • Fixed button to clear integer input

    Clicking in the text box (In lieu of the stepper control) provides a gray x button which clears the input field, however it did not clear the data. Now the clear button returns the text value to its default, which could be nil or 0, but it could also be any other integer value. Learn more...

  • Supported importing of surveys

    Old surveys or surveys from other iPads can be added by putting the *.obssurv folder (inside the *.poz file) in the documents directory in iTunes, or by opening a *.poz file from email or Safari. I cannot import *.poz files put in the iTunes folder, because that is where the exports are.

  • Removed Angle/Distance settings from the settings app.

    The settings can only be defined in the protocol file, which was best, and always used anyways.

  • Support slashes in survey names

    A slash in a survey name as seen as a folder separator, which caused the survey export to fail. Slashes are replaced by underscore in the file name.

  • Auto split long long tracklogs at 1000 gps points

    This prevents very very long polylines from getting created when the user does not stop observing. The long polylines could slow down input and drawing. The split tracklogs have all the same attributes, and will appear the same in ArcMap, and can be merged if needed.

  • Add Cancel button to attribute editing form

    Default when dismissing the form is to save all edits. This will close the form and discard edits. No warning is given.

  • Observation selector uses label field

    When you tap to select a feature, and multiple feature are available, you need to pick the feature you want from a list. Previously, you had to know the creation date to choose the one you wanted. Now if you have a your feature has a label, you can use the label to select the feature you want.

  • Numerous bug fixes

    Survey race conditions; Crashes with map loading, defaults not being recognized after upgrade Move to GPS point not always updating the display. Duplicate graphics for a single feature. Crashed if you started data logging without a map. Deleting any map incorrectly cleared the map display. And lots more.

  • Modernization of code for latest iOS versions

    Lots of code cleanup and refactoring to support best practices for modern iOS code. Turned on all compiler warnings and eliminated all warnings. Removed dead and un-needed code.
